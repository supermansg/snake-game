# AGENTS.md

## Mission
Improve this project in small, verified, high-confidence steps.

## Rules
- Always inspect the current structure before making changes.
- Prefer minimal changes with clear impact.
- Preserve working behavior unless improvement is explicitly required.
- Prioritize mobile responsiveness and clean UI.
- Do not leave overlapping controls or broken gameplay states.

## Required validation
Before finishing any meaningful task, run:
1. npm install if needed
2. npm run lint
3. npm run build

## Skills guidance
- Use game-development for gameplay and polish improvements.
- Use systematic-debugging for bugs and root-cause analysis.
- Use web-design-guidelines for layout, UI, and mobile fixes.
- Use code-review-excellence before concluding major feature work.
- Use deploy-to-vercel when preparing release-ready changes.
- Use agent-browser for browser-based QA and flow validation.

## Completion format
Always finish with:
- What changed
- Files changed
- Validation run
- Remaining risks
- Recommended next step
